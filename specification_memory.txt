1. ЗАГАЛЬНИЙ ОПИС
────────────────────────────────────────
Контролер пам'яті КВ-32 забезпечує інтерфейс між 32-розрядним процесором
та підсистемою пам'яті, яка складається з:
  — ОЗП: 2 банки по 2К×32 = 4К слів × 32 біти = 16 КБ
  — ПЗП: 2 банки по 8К×32 = 16К слів × 32 біти = 64 КБ
Загальний адресний простір: 80 КБ (адреси 0x0000–0xFFFF, 16-бітна адреса)

Карта адресного простору:
  0x0000–0x07FF  (2КБ)  CS0 → ОЗП Банк 1
  0x0800–0x0FFF  (2КБ)  CS1 → ОЗП Банк 2
  0x2000–0x3FFF  (8КБ)  CS2 → ПЗП Банк 1 (завантажувальний код)
  0x4000–0x5FFF  (8КБ)  CS3 → ПЗП Банк 2

Декодування: A[15:13] → К1533ИД7 (3→8 дешифратор):
  A[15:13] = 000 → CS0 (ОЗП Б1)
  A[15:13] = 001 → CS1 (ОЗП Б2)
  A[15:13] = 010 → CS2 (ПЗП Б1)
  A[15:13] = 011 → CS3 (ПЗП Б2)
  A[15:13] = 100–111 → NC (не використовується)

2. СКЛАД БЛОКУ ТА ПРИЗНАЧЕННЯ МІКРОСХЕМ
────────────────────────────────────────
2.1. DD1 — К1533ИД7 (Дешифратор адреси)
     Аналог: 74LS138 (3→8 дешифратор з дозволами)
     Корпус: DIP-16
     Призначення: декодування старших адресних біт A[15:13] для формування
     чотирьох сигналів вибору мікросхем /CS0–/CS3 (активний рівень LOW).

2.2. DD2–DD5 — К537РУ8 (ОЗП Банк 1, 4 мікросхеми)
     Статична SRAM 2048×8 біт = 2К×32 (4 паралельних мікросхеми)
     Корпус: DIP-24
     Призначення: оперативна пам'ять нижнього банку, молодша частина
     адресного простору (0x0000–0x07FF).

2.3. DD6–DD9 — К537РУ8 (ОЗП Банк 2, 4 мікросхеми)
     Аналогічно DD2–DD5, верхній банк ОЗП (0x0800–0x0FFF).

2.4. DD10–DD13 — К596РЕ1 (ПЗП Банк 1, 4 мікросхеми)
     EPROM 8192×8 біт = 8К×32 (4 паралельних мікросхеми)
     Корпус: DIP-28
     Призначення: постійна пам'ять програм завантаження (0x2000–0x3FFF).

2.5. DD14–DD17 — К596РЕ1 (ПЗП Банк 2, 4 мікросхеми)
     Аналогічно DD10–DD13 (0x4000–0x5FFF).

5. ЧАСОВІ ПАРАМЕТРИ ТА ЦИКЛОГРАМА ШИНИ
────────────────────────────────────────
5.1. К537РУ8 (SRAM):
  t_access (/CS або A → D_out)  = 120 нс (max)
  t_ce     (/CS → D_out)         = 120 нс (max)
  t_wr     (/WE LOW тривалість)  = 70 нс (min)
  t_setup  (D_in → /WE HIGH)     = 0 нс
  t_hold   (після /WE HIGH)      = 10 нс

5.2. К596РЕ1 (EPROM):
  t_access (A → O_out)           = 200 нс (max)
  t_ce     (/CE → O_out)         = 200 нс (max)
  t_oe     (/OE → O_out, з стану HiZ) = 75 нс (max)

5.3. Циклограма читання (f_CLK = 5 МГц, T = 200 нс):
  ОЗП (К537РУ8):  1 такт = 200 нс;  t_access=120 нс; запас=80 нс ✓
  ПЗП (К596РЕ1): 2 такти = 400 нс; t_access=200 нс; запас=200 нс ✓
    → Для ПЗП потрібен 1 такт очікування (WAIT state)
    → Процесор подовжує цикл шини: MREQ залишається активним

5.4. Циклограма запису в ОЗП (1 такт):
  T1 фронт: A[15:0] → CS (через K1533ИД7 ≈ 20 нс) → CS активний
  T1 середина: /WR → LOW (запис активний)
  T1 спад: D[31:0] → шина даних (t_setup ≥ 0 нс від /WR HIGH)
  T1 кінець: /WR → HIGH (дані захоплені в SRAM)

6. ТОПОЛОГІЯ ШИНИ ДАНИХ
────────────────────────────────────────
32-бітна шина даних DB[31:0] є двоспрямованою для ОЗП і односпрямованою
для ПЗП (тільки читання). Всі 32 лінії підключені до відповідних
виводів D[7:0] паралельно включених мікросхем:

  DB[7:0]   ↔ DD2.D[7:0]  / DD6.D[7:0]  / DD10.O[7:0] / DD14.O[7:0]
  DB[15:8]  ↔ DD3.D[7:0]  / DD7.D[7:0]  / DD11.O[7:0] / DD15.O[7:0]
  DB[23:16] ↔ DD4.D[7:0]  / DD8.D[7:0]  / DD12.O[7:0] / DD16.O[7:0]
  DB[31:24] ↔ DD5.D[7:0]  / DD9.D[7:0]  / DD13.O[7:0] / DD17.O[7:0]

Одночасно активна лише одна група (або ОЗП Б1, або ОЗП Б2, або ПЗП Б1,
або ПЗП Б2) завдяки дешифрування /CS0–/CS3.

7. СХЕМА РОЗПОДІЛУ ЖИВЛЕННЯ
────────────────────────────────────────
Напруга живлення: +5В ± 5%
Споживання (орієнтовно):
  DD1 К1533ИД7:         I_cc ≈ 10 мА
  DD2–DD9 К537РУ8:      8 × 40 мА = 320 мА
  DD10–DD17 К596РЕ1:    8 × 55 мА = 440 мА
  Всього: ~770 мА ≈ 0.8 А

Розв'язувальні конденсатори:
  C1–C17 = 100 нФ (X7R, 50В) — по одному між VCC–GND кожної мікросхеми
  C18 = 10 мкФ/16В (електролітичний) — груповий по +5В шині
  Розміщення: впаяти якомога ближче до ніжок VCC–GND

Підтягувальні резистори /OE:
  R1=1кОм: /OE ОЗП Б1 (4 паралельно, еквівалент навантаження: 0.25 мА)
  R2=1кОм: /OE ОЗП Б2
  R3=1кОм: /OE ПЗП Б1
  R4=1кОм: /OE ПЗП Б2
  Мета: уникнення невизначеного стану на виходах при деактивованому /OE

8. ОБОВ'ЯЗКОВІ ЗВ'ЯЗКИ НА СХЕМІ (ЄСКД)
────────────────────────────────────────
8.1. Адресна шина A[15:0]:
  ✓ A[15:13] → DD1 C,B,A (декодування CS)
  ✓ A[12:0]  → DD10–DD17 A[12:0] (ПЗП)
  ✓ A[10:0]  → DD2–DD9 A[10:0] (ОЗП)

8.2. Шина даних DB[31:0] (двоспрямована для ОЗП):
  ✓ DB[7:0]   ↔ DD2, DD6, DD10, DD14 D/O[7:0]
  ✓ DB[15:8]  ↔ DD3, DD7, DD11, DD15 D/O[7:0]
  ✓ DB[23:16] ↔ DD4, DD8, DD12, DD16 D/O[7:0]
  ✓ DB[31:24] ↔ DD5, DD9, DD13, DD17 D/O[7:0]

8.3. Сигнали вибору мікросхем:
  ✓ /CS0 → DD2–DD5 /CS
  ✓ /CS1 → DD6–DD9 /CS
  ✓ /CS2 → DD10–DD13 /CE
  ✓ /CS3 → DD14–DD17 /CE

8.4. Керуючі сигнали:
  ✓ /WR → DD2–DD9 /WE (тільки ОЗП, ПЗП не записуємо)
  ✓ /RD → R1–R4 знімає /OE з усіх активних мікросхем
  ✓ /MREQ → DD1 /G2B (дозвіл дешифратора)

8.5. Живлення:
  ✓ +5В рейка → VCC всіх мікросхем + C1–C18
  ✓ GND рейка → GND всіх мікросхем
  ✓ XP1 — роз'єм живлення на границі блоку
